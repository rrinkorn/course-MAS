# Курс: Инженерия AI-агентов и мультиагентных систем

## Модуль 1: Введение в AI-агенты

**Цель:** Сформировать фундаментальное понимание агентности, отличий от классического NLP и архитектурных принципов построения автономных систем.

1.  **От NLP к агентам — Путешествие длиной в полвека** *(from_nlp_to_agents.md)*
    *   Эпоха правил (1950-1990): ELIZA, экспертные системы, MYCIN.
    *   Статистическая революция (1990-2010): n-граммы, наивный байесовский классификатор.
    *   Нейронные сети (2010-2017): Word2Vec, LSTM, encoder-decoder архитектуры.
    *   Революция трансформеров (2017): механизм внимания, BERT vs GPT.
    *   Масштабирование (2018-2019): законы масштабирования, эмерджентные способности.
    *   Instruct Tuning и RLHF: как модели научились следовать инструкциям.
    *   Год агентов (2023): плагины, AutoGPT, BabyAGI, паттерн ReAct.
2.  **Анатомия AI-агента** *(agent_anatomy.md)*
    *   Три ключевых свойства агента: автономность, целенаправленность, восприятие/воздействие.
    *   Цикл OODA (Observe-Orient-Decide-Act) в контексте LLM.
    *   Четыре компонента агента: LLM (мозг), Tools (руки), Memory (хранилище), Planning (стратегия).
    *   Типы памяти: рабочая, эпизодическая, семантическая, процедурная.
    *   Свойства среды: полнота наблюдаемости, детерминированность, статичность.
    *   Полный цикл работы агента из 8 шагов.
3.  **Когнитивные архитектуры AI-агентов** *(cognitive_architectures.md)*
    *   System 1 и System 2 мышления (Канеман) — быстрое интуитивное и медленное аналитическое.
    *   Четыре столпа архитектуры: Profile, Memory, Planning, Action (PMPA).
    *   ReAct (Reasoning + Acting): циклическое чередование мысли и действия.
    *   LATS (Language Agent Tree Search): древовидный поиск с оценкой альтернатив.
    *   AutoGPT и BabyAGI: амбициозные попытки полной автономности.
    *   Сравнение архитектур по сложности, надёжности, стоимости.
4.  **Паттерны рассуждений** *(reasoning_patterns.md)*
    *   Zero-shot и Few-shot prompting.
    *   Chain of Thought (CoT): "Let's think step by step".
    *   ReAct как паттерн промптинга (Thought → Action → Observation).
    *   Tree of Thoughts (ToT): ветвление и исследование альтернатив.
    *   Graph of Thoughts (GoT): обобщение на произвольные графы.
    *   Self-Consistency: множественные запуски с majority voting.
    *   Reflection: самокритика и исправление ошибок.
5.  **Ограничения современных LLM** *(llm_limitations.md)*
    *   Галлюцинации: фактические, контекстные, логические, выдуманные источники.
    *   Контекстное окно: проблема "Lost in the Middle".
    *   Knowledge cutoff: модель не знает о событиях после даты обучения.
    *   Проблемы математики и логики.
    *   Долгосрочное планирование: близорукость, невыполнимые планы.
    *   Консистентность и детерминизм.
    *   Методы митигации: RAG, верификация, Code Interpreter, Reflection.
6.  **Экосистема и экономика AI-агентов** *(ecosystem_economics.md)*
    *   Провайдеры: OpenAI, Anthropic, Google, Mistral, DeepSeek, Qwen.
    *   Open source лидеры: Qwen 3, DeepSeek-V3, Mistral, Llama.
    *   Русскоязычные модели: GigaChat, YandexGPT, Saiga.
    *   Инструменты локального запуска: Ollama, LM Studio, vLLM.
    *   Ценообразование: input vs output токены, сравнение цен.
    *   Расчёт TCO (Total Cost of Ownership) агента.
    *   Оптимизация стоимости: каскадирование, кэширование, батчинг.
    *   ROI агентов и матрица выбора модели.
7.  **Бенчмарки и оценка качества LLM** *(benchmarks.md)*
    *   Анатомия бенчмарка: датасет, формат задач, метрика, протокол.
    *   Категории бенчмарков: общие знания (MMLU, ARC-AGI), математика (MATH, AIME), программирование (HumanEval, SWE-bench), инструкции (IFEval), мультимодальность (MMMU), агентные задачи (OSWorld, WebArena, GAIA).
    *   Сравнение топовых моделей по бенчмаркам.
    *   Подводные камни: насыщение, data contamination, overfitting.
    *   Ресурсы: LMSYS Chatbot Arena, Artificial Analysis, HF Leaderboard.

## Модуль 2: LangChain

**Цель:** Изучить стандарт де-факто для оркестрации LLM приложений, освоить декларативный язык выражений LCEL.

1.  **Философия LangChain и основы LCEL** *(philosophy_and_lcel_basics.md)*
    *   История развития LangChain (октябрь 2022 — 2024).
    *   Архитектура: langchain-core, интеграции, langchain, langchain-community, langgraph.
    *   Философия композиции как фундамент.
    *   Интерфейс Runnable: invoke, batch, stream, асинхронные версии.
    *   Синтаксис Pipe `|` и его аналогия с Unix.
    *   Примитивы: RunnableSequence, RunnableParallel, RunnablePassthrough, RunnableLambda.
    *   Поток данных и трансформации типов.
    *   Визуализация и отладка (get_graph(), трейсинг, LangSmith).
2.  **LCEL — продвинутые паттерны** *(lcel_advanced_patterns.md)*
    *   Динамическая маршрутизация: RunnableBranch, RunnableRouter, семантическая маршрутизация.
    *   Устойчивость к сбоям: with_retry(), with_fallbacks(), обработка исключений.
    *   Стриминг: stream(), astream(), astream_events().
    *   Эффективность: батчинг (batch(), abatch()) и конкурентность.
    *   Конфигурация времени выполнения: RunnableConfig, ConfigurableField, configurable_alternatives.
    *   Callbacks и наблюдаемость: BaseCallbackHandler.
    *   Кэширование: InMemoryCache, SQLiteCache, RedisCache, semantic cache.
3.  **Управление промптами — искусство разговора с моделью** *(prompt_engineering.md)*
    *   Анатомия сообщений: роли System, Human, AI, Function/Tool.
    *   Классы сообщений: SystemMessage, HumanMessage, AIMessage.
    *   Мультимодальные сообщения: текст + изображения, base64.
    *   ChatPromptTemplate: создание и форматирование шаблонов.
    *   Few-shot prompting: примеры в промптах, FewShotChatMessagePromptTemplate.
    *   Динамический выбор примеров: SemanticSimilarityExampleSelector.
    *   MessagesPlaceholder: динамическая вставка истории, optional placeholders.
    *   Управление длиной истории: trim_messages.
    *   Partial formatting: частичное заполнение переменных.
    *   Композиция промптов: объединение, PipelinePromptTemplate.
    *   Best practices и антипаттерны.
4.  **Модели и Model I/O** *(models_and_io.md)*
    *   Два мира моделей: LLM (completion) vs ChatModel (диалог).
    *   Экосистема провайдеров: OpenAI, Anthropic, Google, локальные через Ollama.
    *   Параметры генерации: Temperature, Top-P, Max Tokens, Stop Sequences, Seed.
    *   Отслеживание использования: подсчёт токенов, callbacks, мониторинг.
    *   Экономика токенов: цены по провайдерам и моделям.
    *   Мультимодальность: работа с изображениями, base64.
    *   Потоковая генерация: stream(), astream(), astream_events().
    *   Интеграция с LCEL: bind(), configurable_fields, configurable_alternatives.
5.  **Структурированный вывод и Output Parsers** *(output_parsers.md)*
    *   Философия структурированного вывода: JSON как lingua franca.
    *   Два пути к структуре: инструкции в промпте vs структурированные API.
    *   Базовые парсеры: StrOutputParser, JsonOutputParser, PydanticOutputParser, CommaSeparatedListOutputParser.
    *   with_structured_output(): constrained decoding, json_mode vs function_calling, strict mode.
    *   Обработка ошибок парсинга: OutputFixingParser, RetryWithErrorOutputParser, fallback-стратегии.
    *   Продвинутые паттерны: вложенные схемы Pydantic, Optional/Union типы, Enum, кастомные валидаторы.
    *   Streaming с парсерами.
    *   Практические рецепты: извлечение сущностей, классификация, пошаговый анализ.
6.  **Память и Цепочки — от компонентов к приложениям** *(memory_and_chains.md)*
    *   Проблема памяти: stateless-природа LLM, контекстное окно, Lost in the Middle.
    *   Стратегии управления памятью: Buffer Memory, Summary Memory, Entity Memory, Hybrid.
    *   Реализация памяти в LCEL: история как входной параметр, явное управление.
    *   RunnableWithMessageHistory для автоматизации.
    *   Персистентные хранилища: Redis, PostgreSQL.
    *   Классические цепочки (Legacy): LLMChain, ConversationChain, SequentialChain, MultiPromptChain, Router Chains.
    *   Практические паттерны: чат-бот с RAG и памятью, мультиагентная обработка, самокорректирующаяся генерация.
    *   Миграция на LangGraph.

**Практика: "Умный ассистент с характером"**
*   **Задача:** Создать CLI-бота, который принимает команду на естественном языке, классифицирует её (спам/запрос/жалоба) и отвечает в строго заданном JSON формате.
*   **Инструменты:** LangChain, Pydantic, LCEL.
*   **Дополнительно:** Реализовать стриминг ответа, кэширование и fallback на дешёвую модель.

## Выпускной проект

**Обязательные требования:** LangGraph, Память, Минимум 2 инструмента, README с архитектурой.

### Примеры проектов

#### Категория: Бизнес и Документы
1.  **AI Legal Auditor (Юрист)**
    *   *Задача:* Автоматическая проверка договоров на риски.
    *   *Вход:* PDF договор.
    *   *Процесс:* OCR -> RAG по ГК РФ/Базе рисков -> Выделение пунктов -> Классификация риска (High/Medium/Low).
    *   *Результат:* Таблица с правками и комментариями в формате DOCX.
2.  **Smart HR Recruiter**
    *   *Задача:* Скрининг резюме и первичный контакт.
    *   *Вход:* Архив PDF резюме + Описание вакансии.
    *   *Процесс:* Парсинг -> Матчинг с вакансией (0-100%) -> Генерация персонализированных вопросов для интервью -> Отправка Email (Mock).
    *   *Результат:* JSON список кандидатов с ранжированием и лог рассылки.
3.  **Tender Procurement Assistant**
    *   *Задача:* Анализ тендерной документации.
    *   *Вход:* Ссылка на закупку (Zip архив документов).
    *   *Процесс:* Скачивание -> Чтение ТЗ -> Сравнение с профилем компании -> Оценка маржинальности -> Генерация черновика заявки.
    *   *Результат:* "Go/No-Go" решение и список недостающих документов.
4.  **Financial Report Analyst**
    *   *Задача:* Анализ квартальных отчетов (Excel/PDF).
    *   *Вход:* Финансовые отчеты за 3 года.
    *   *Процесс:* Извлечение таблиц -> Расчет коэффициентов (P/E, EBITDA) через Python Code Interpreter -> Сравнение год-к-году -> Написание выводов.
    *   *Результат:* Аналитическая записка с графиками трендов.

#### Категория: Разработка и IT
5.  **GitHub Issue Solver Agent**
    *   *Задача:* Авто-фикс багов.
    *   *Вход:* Ссылка на Issue в GitHub.
    *   *Процесс:* Клон репо -> Анализ кода -> Написание теста, воспроизводящего баг -> Исправление кода -> Прогон теста -> Создание PR.
    *   *Результат:* Ссылка на Pull Request.
6.  **DevOps Log Sentinel**
    *   *Задача:* Анализ логов серверов при инциденте.
    *   *Вход:* Сырой лог файл (Nginx/App errors).
    *   *Процесс:* Кластеризация ошибок -> Поиск решений в StackOverflow/Docs (Search Tool) -> Предложение команд для фикса.
    *   *Результат:* Отчет Root Cause Analysis (RCA).
7.  **Legacy Code Migrator**
    *   *Задача:* Перевод кода с одного языка на другой (напр., Java -> Python).
    *   *Вход:* Папка с исходным кодом.
    *   *Процесс:* Построение графа зависимостей -> Пофайловая трансляция -> Адаптация идиом языка -> Генерация Unit-тестов для проверки эквивалентности.
    *   *Результат:* Новый репозиторий с мигрированным кодом.
8.  **Automated QA Engineer**
    *   *Задача:* Генерация E2E тестов.
    *   *Вход:* URL веб-страницы + описание пользовательского сценария.
    *   *Процесс:* Анализ DOM-дерева -> Генерация кода Playwright/Selenium -> Запуск теста -> Если упал, само-коррекция селекторов.
    *   *Результат:* Python скрипт теста и видео прогона.

#### Категория: Обучение и Наука
9.  **Personalized Learning Tutor**
    *   *Задача:* Обучение сложной теме.
    *   *Вход:* Тема (например, "Квантовая механика") + уровень студента.
    *   *Процесс:* Генерация плана -> Поиск материалов -> Генерация лекции -> Квиз -> Анализ ошибок студента -> Адаптация следующей лекции.
    *   *Результат:* Интерактивный Telegram-бот с сохранением прогресса.
10. **Research Paper Assistant (Arxiv)**
    *   *Задача:* Глубокий обзор литературы.
    *   *Вход:* Научный вопрос.
    *   *Процесс:* Поиск статей на Arxiv -> Скачивание PDF -> Выделение гипотез и методов -> Построение графа цитирований -> Синтез мета-обзора.
    *   *Результат:* Markdown-статья с библиографией.
11. **Language Practice Buddy**
    *   *Задача:* Практика иностранного языка.
    *   *Вход:* Голосовой ввод.
    *   *Процесс:* Поддержание диалога (Roleplay) -> STT -> Анализ грамматических ошибок "на лету" -> Мягкая коррекция в ответе -> TTS.
    *   *Результат:* Голосовой диалог и "карточка ошибок" в конце сессии.

#### Категория: Лайфстайл и Личное
12. **AI Travel Agent (End-to-End)**
    *   *Задача:* Полное планирование путешествия.
    *   *Вход:* "Хочу в Японию весной на 2 недели, бюджет $3000".
    *   *Процесс:* Поиск билетов (Mock API) -> Поиск отелей -> Составление маршрута по дням с учетом погоды и локаций -> Расчет бюджета.
    *   *Результат:* PDF маршрут с ссылками на бронирование.
13. **Smart Nutritionist**
    *   *Задача:* Планирование питания по фото.
    *   *Вход:* Фото продуктов в холодильнике + цели (похудение/набор).
    *   *Процесс:* Vision API (распознавание продуктов) -> Поиск рецептов -> Составление меню на неделю -> Генерация списка покупок недостающего.
    *   *Результат:* План питания в Telegram.
14. **Personal Finance Auditor**
    *   *Задача:* Анализ личных расходов.
    *   *Вход:* CSV выписка из банка.
    *   *Процесс:* Категоризация транзакций (ML + Rules) -> Выявление аномалий и подписок -> Советы по оптимизации.
    *   *Результат:* Дашборд и советы.

#### Категория: Медиа и Маркетинг
15. **Content Repurposing Factory**
    *   *Задача:* Превращение видео в статьи и посты.
    *   *Вход:* Ссылка на YouTube видео.
    *   *Процесс:* Транскрибация -> Саммаризация -> Написание лонгрида -> Написание треда для Twitter -> Генерация обложки.
    *   *Результат:* Пакет контента для разных соцсетей.
16. **Competitor Watchdog**
    *   *Задача:* Мониторинг конкурентов.
    *   *Вход:* Список сайтов конкурентов.
    *   *Процесс:* Ежедневный обход -> Детекция изменений (цены, новые баннеры) -> Анализ тональности отзывов -> Сводка новостей.
    *   *Результат:* Ежедневный Email-отчет.
17. **Trend Spotter**
    *   *Задача:* Поиск вирусных тем.
    *   *Вход:* Ключевые слова.
    *   *Процесс:* Скан Reddit/Twitter/News -> Кластеризация тем -> Оценка потенциала "вирусности" -> Генерация идей для контента.
    *   *Результат:* Список идей с обоснованием.

18. **Smart Home Butler**
    *   *Задача:* Управление умным домом языком.
    *   *Вход:* Голосовая команда ("У меня свидание, сделай красиво").
    *   *Процесс:* Интерпретация интента -> n8n Webhook -> API умного дома (свет, музыка) -> Заказ еды (Mock).
    *   *Результат:* Выполнение серии команд IoT.

    *   *Задача:* Контроль качества поддержки.
    *   *Вход:* История тикетов Zendesk/Intercom.

    *   *Результат:* Рейтинг операторов и алерты.
20. **Meeting Minutes Assistant**
    *   *Задача:* Авто-протоколирование встреч.
    *   *Вход:* Аудиозапись встречи.
    *   *Процесс:* Диаризация (кто говорит) -> Транскрибация -> Выделение Action Items -> Создание задач в Jira/Trello/{analog} (через n8n).
    *   *Результат:* Задачи в таск-трекере и письмо участникам.
